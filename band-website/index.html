The mental model

How a website actually works

  • A website is just a folder of files. Really. Text files that a browser knows how to draw.
  • HTML is the content — your words, your headings, your photos.
  • CSS is the look — colors, fonts, spacing, the vibe.
  • GitHub Pages takes your folder and hands it a public address anyone can visit.

You edit the folder → you upload the folder → GitHub shows it to the world. That's the entire loop.

Your project folder

The file structure

Make one folder on your computer. Inside it, this:

my-band-site/ ├── index.html # your home page ├── style.css # the look & colors └── images/ # band photos go here └── band.jpg

index.html is special — GitHub always shows it first. It IS your homepage.

Step 1 · The page

Your home page

In VS Code, make a file called index.html and type this. Swap in your band's real words.

index.html 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>The Wildflowers</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1>The Wildflowers</h1>
  <p>Folk-rock from Boston. Loud when it counts.</p>
  <img src="images/band.jpg" alt="The band">
  <h2>Shows</h2>
  <p>Aug 14 — The Sinclair, Cambridge</p>
  <p>Email: hello@wildflowers.com</p>
</body>
</html>

Step 3 · See it before the world does

Test it on your own machine

  • Find your index.html file in your folder.
  • Double-click it. It opens in your browser.
  • That's your website — running right now, just privately on your laptop.
  • Don't like something? Edit in VS Code, save, refresh the browser. Repeat until it's you.

Nothing is public yet. This is your rehearsal space. Get it right here first.

Step 4 · Get it online

Upload to GitHub

No terminal. No commands. Just drag and drop.

  • On github.com, click the + (top right) → New repository.
  • Name it your-band-site. Set it to Public. Click Create.
  • On the new page, click "uploading an existing file."
  • Drag your whole folder's contents in (index.html, style.css, images). Click Commit changes.

"Repository" is just GitHub's word for your project folder. That's all it means.

Step 5 · Flip the switch

Turn on GitHub Pages

  • In your repository, click Settings (top menu).
  • On the left sidebar, click Pages.
  • Under "Branch," pick main, keep the folder as / (root), click Save.
  • Wait about a minute. Refresh. GitHub shows you a green link.

That green link is your live website. Anyone on earth can open it.

You're live

Your public address

Every site on GitHub Pages follows the same pattern. Yours becomes:

yourname.github.io/your-band-site

Put it on your Instagram bio. Text it to a venue. It's real and it's yours.

The part that matters most

Updating it later

  • New show? New photo? Open the file in VS Code, change the words, save.
  • Go to your repo on github.com → click the file → click the pencil ✏️ → paste your changes → Commit.
  • Or drag a whole new version in, same as before.
  • The live site updates itself in about a minute. Every single time. Free.

This is the whole reason to own it yourself: you're never stuck waiting on anyone.

Bring in outside media

Embed a YouTube video

GitHub Pages never has to host the video file itself — you just embed a window into YouTube's copy.

This is a real, live embed — not a screenshot. Same code you're about to copy.

  • On YouTube: Share → Embed gives you this code.
  • VIDEO_ID is the part of the URL after watch?v=.
  • &t=49s in a normal link becomes ?start=49 here — delete the ?si=... tracking code YouTube adds, it isn't needed.
  • Delete the width="560" height="315" YouTube bakes in — the CSS wrapper controls the size instead.
index.html 
<div class="video-wrap">
  <iframe src="https://www.youtube.com/embed/VIDEO_ID?start=49"
    title="Band video"
    allow="accelerometer; autoplay;
      encrypted-media; gyroscope; picture-in-picture"
    referrerpolicy="strict-origin-when-cross-origin"
    allowfullscreen></iframe>
</div>
style.css 
/* keeps any video the right shape on any screen */
.video-wrap {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 */
  height: 0;
}
.video-wrap iframe {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
}
When you get stuck

The rules of not panicking

  • Every open tag needs a close tag. <h1> needs </h1>. That's 90% of bugs.
  • File names are exact. style.css is not Style.css. images/band.jpg is not images/Band.JPG.
  • Save the file before you refresh. VS Code shows a dot when unsaved.
  • Change one thing, then look. Don't change ten things and wonder which broke it.

You don't need an AI to write this for you. You need to close your tags and save your file.
